2.关机重启

·基本介绍

​ shutdown -h now 立刻进行关机

​ shutdown -h 1 一分钟之后关机

​ shutdown -r now 现在重启计算机

​ halt 同上

​ reboot 现在重启计算机

​ sync 把内存的数据同步到磁盘

·注意细节

​ 1.不管是重启还是关闭系统，首先要执行sync命令。

​ 2.目前的shutdown halt reboot 都已经执行了sync命令

​ logout 是注销用户

4.用户管理

添加用户

  • 基本语法

    ​ useradd

    passwd 用户名–指定/修改密码 pwd当前用户所在的目录

用户组

​ 介绍：系统可以对有共性/权限的多个用户进行统一的管理.

​ 新增组：groupadd 组名

​ 删除组：groupdel 组名

​ 增加用户时直接加上组： useradd -g 用户组 用户名

​ 修改用户的组：usermod -g 用户组 用户名

/etc/passwd 文件

​ 用户的配置文件，记录用户的各种信息

​ 每行的含义：用户名：口令：用户标识号：组标识号：注释性描述：主目录：登录SHELL

/etc/shadow文件

​ 口令的配置文件

​ 每行的含义：登录名：加密口令：最后一次修改时间：最小的时间间隔：最大的时间间隔：警告时间：不活动时间：失效时间：标志

/etc/group文件

​ 组的配置文件，记录Linux包含的组的信息

​ 每行含义：组名：口令：组标识号：组内的用户列表

修改密码

​ passwd 用户名

删除用户

​ userdel 用户名

​ 保留家目录 userdel 用户名

​ 删除用户以及用户主目录 userdel -r 用户名

切换用户

​ su 用户名

查看当前用户

Spring Boot 实现 RESTful API

SpringBoot 提供的spring-boot-starter-web组件完全支持开发RESTful API,

提供了与REST操作方式(GET POST PUT DELETE)对应的注解。

@GetMapping：处理GET请求，获取资源。

@PostMapping: 处理POST请求，新增资源。

@PutMapping:处理PUT请求，更新资源。

@DeleteMapping：处理DELETE请求，删除资源。

@PatchMapping：处理PATCH请求，用于部分更新资源.

Http 协议

请求协议 响应协议 协议解析

特点：1.基于TCP协议 面向连接 安全

​ 2.基于请求-响应模型的 一次请求对应一次响应

​ 3.HTTP协议是无状态的协议 对于事务处理没有记忆能力 每次请求响应都是独立的

​ 缺点：多次请求间不能共享数据

​ 优点：速度快
